解决日记：Argon 主题小工具「年份/季节进度条」与自定义鼠标光标（2026-01-01）

今天做了什么

在 WordPress（Argon Theme + PJAX）里做了两个前端小功能，并排查了「进度条不动/不显示」等问题：

  1. 自定义鼠标光标：桌面端显示，手机/平板不显示。
  2. 年份进度条 + 季节进度条：季节进度在冬季跨年时计算错误，且一度出现 UI 不显示（脚本报错）。

问题 1：自定义鼠标光标在网页变窄时消失

现象

鼠标光标使用 @media (max-width: 1300px) 控制隐藏，导致浏览器窗口变窄时也隐藏（即使仍在桌面端）。

原因

max-width 是按 视口宽度 判断，不是按“设备类型”。桌面浏览器窗口变窄也会命中媒体查询。

解决思路

用「设备能力」而不是「宽度」判断：

  • 通过 CSS 判断是否支持 hover（大多数桌面鼠标支持 hover）。
  • 或用 JS 判断触摸能力（更通用）。

    这里最终目标是：只在桌面端显示，手机/平板不显示。

问题 2：季节进度条「不动」/ 冬季进度算错

现象

春夏秋正常，冬季（12月～次年2月）会出现进度一直 0% 或跳变异常。

根因

Date.getMonth() 返回 0~11（0 代表 1 月）。citeturn0search0
冬季跨年时，如果只把 seasonYear--（把年份挪到上一年），却仍用同一个年份去算 2 月的结束日期，会导致：

  • seasonEnd < seasonStart（结束时间比开始时间还早）
  • 进度变负数后又被 clamp 到 0 → 看起来“不动”

解决方法

冬季要拆成两个年份：

  • 12月：开始年 = 当年，结束年 = 次年
  • 1-2月：开始年 = 上一年，结束年 = 当年

另外，计算“本月最后一天”可用常见写法：
new Date(year, month + 1, 0)（day=0 表示“上个月最后一天”，因此 month+1 的 0 天就是“本月最后一天”）。citeturn0search6

问题 3：UI 不显示 + 控制台报错 Unexpected token '<'

现象

进度条 UI 空白，控制台出现：

  • Uncaught SyntaxError: Unexpected token '<'

根因（常见两种）

  1. JS 里混入了 HTML：例如把 <div>...</div> 或嵌套 <script> 放进了 JS 上下文，解析器遇到 < 就报错。
    MDN 对 “Unexpected token” 的说明：解析器在当前位置遇到不符合语法的标记会抛错。citeturn0search3
  2. 请求到的“JS 文件”实际返回了 HTML（比如 404 页面 / 重定向页），同样会以 <html... 开头导致报错（典型现象）。citeturn0search7

排查步骤（当时很有效）

  • 看 console 第一条红错，确认是否是 Unexpected token '<'。
  • 验证 jQuery 是否存在：window.jQuery。
  • 验证 DOM 元素是否存在：document.getElementById("yearprogress_yearname")。
  • 手动写入测试：document.getElementById("yearprogress_yearname").textContent="TEST"
    若能显示 TEST，说明是“初始化没执行/报错中断”，而不是 CSS 隐藏。

额外注意：WordPress 可能过滤 <script>

WordPress 文档说明：如果用户缺少 unfiltered_html 权限，系统会用 wp_kses() 清理内容，<script> 之类标签会被剥离。citeturn2search1
（本次最终确认脚本能保留且 jQuery 存在，所以不是这个原因，但记录在这里方便以后排查。）

最终可用版本：只改小工具即可（修复冬季跨年 + 防止 interval 叠加）

放在 外观 → 小工具 → 自定义 HTML（允许 HTML/CSS/JS 的位置）。

经验总结（下次遇到类似问题怎么快查）

  • 先看 Console 第一条红错：Unexpected token '<' 往往就是“JS 里混了 HTML”或“加载到的 JS 实际是 HTML”。citeturn0search3turn0search7
  • 时间相关逻辑先确认：
    • getMonth() 是 0~11（0=1月）。citeturn0search0
    • 跨年区间不要只改一个年份变量，开始/结束年要分开算。
  • WordPress 里 <script> 被吃掉：优先检查是否有 unfiltered_html。citeturn2search1
  • 需要算月末：new Date(y, m+1, 0) 是最简洁常用法。citeturn0search6
